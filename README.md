# さんごう防災3Dマップ

奈良県三郷町の [PLATEAU 3D都市モデル (2025年度)](https://www.geospatial.jp/ckan/dataset/plateau-29343-sango-cho-2025) を活用した、住民向け防災Webアプリです。

## 解決する自治体課題

三郷町は大和川と竜田川の合流部に市街地が広がり**洪水浸水リスク**を抱える一方、信貴山麓には**土砂災害警戒区域**が多数指定されています。紙のハザードマップは「自分の家がどうなるのか」が直感的に伝わりにくく、避難行動につながりにくいという課題があります。

本アプリは「**タップするだけで、その場所のリスクと逃げ先がわかる**」ことに焦点を絞り、PLATEAUの3D建物モデル上でリスクを自分ごと化します。

## 主な機能

| 機能 | 内容 |
|---|---|
| ワンタップ診断 | 地図をタップすると、その地点の浸水深ランク・土砂災害警戒区域該当・**最寄り避難場所 (方角・距離・徒歩分数・経路リンク)** を表示 |
| 現在地診断 | 位置情報からワンタップで自宅・外出先のリスクを確認 |
| 住所検索 | 地理院ジオコーダで町内の住所へジャンプして診断 |
| 3D表示 | PLATEAU建築物モデル + PLATEAU地形を3D表示 |
| ハザード重畳 | 洪水浸水想定 (想定最大規模)・土石流・急傾斜地・地すべりをチップUIで切替、濃さ調整 |
| 避難場所 | 三郷町公式の避難施設データ (46施設、収容人数・対応災害つき) を表示 |
| 緊急輸送道路・町域界 | PLATEAU関連データセットから実行時取得 (取得不可時は自動で無効化) |
| **建物単位リスク診断** | PLATEAU CityGML属性 (浸水ランク・階数) から建物をリスク別に色分け。「3m以上浸水想定かつ2階建て以下=垂直避難困難」を自動判定 |
| **町全体統計ダッシュボード** | ハザードタイル全域解析による浸水想定面積 (深さ別km²) と、建物別リスク棟数・垂直避難困難棟数を集計表示 — 地域防災計画等の根拠データに |
| **浸水深の3D体感表示** | 浸水深クラスを実際の高さを持つ半透明の水柱として3D表示。地理院標高タイルをクライアントでデコードし (ジオイド補正済み)、建物と水位を直接見比べられる |

スマートフォン最適化済み (ボトムシートUI・セーフエリア対応・44px以上のタッチターゲット)。

## アーキテクチャ

- **Vite + CesiumJS** の静的サイト。サーバー不要、GitHub Pagesにそのままデプロイ可能
- データは実行時に公式配信サービスから取得 (アプリへの大容量同梱なし):
  - 建築物3D Tiles・避難施設・緊急輸送道路・町域界: [PLATEAUデータカタログAPI](https://github.com/Project-PLATEAU/plateau-streaming-tutorial) から市区町村コード `29343` で動的解決 (localStorageに7日キャッシュ)
  - ハザードタイル: 国土地理院 重ねるハザードマップ
  - 地形: PLATEAU-Terrain (Ionトークン未設定・取得不可時は地理院標高タイルに自動フォールバック)
- 避難施設はAPI→同梱データ (`public/data/shelter.geojson`、CRC32検証済み)→地理院全国データの3段フォールバック
- 地点診断はハザードタイルのピクセル色をcanvasで読み取り、公式凡例色と照合して浸水深を判定

## 開発・実行・テスト

```bash
npm install
npm run dev      # 開発サーバー
npm test         # 単体テスト (タイル座標・色判定・距離計算・実データ検証など)
npm run build    # dist/ に静的ビルド
```

mainブランチへのプッシュでGitHub Pagesへ自動デプロイするワークフローを同梱 (`.github/workflows/deploy.yml`)。

## 動作保証の範囲

- **テストで保証**: 座標⇔タイル変換、浸水深の色判定、距離・方位計算、避難所データのパースとフィルタ、同梱避難所データ46件の座標・住所整合性 (`npm test`、10件)
- **設計で担保**: 外部API (PLATEAU配信・地理院タイル) は全系統にフォールバックとエラー表示を実装。パネルの「データ取得状況」で各系統の状態を確認可能
- 開発環境のネットワーク制約により、外部配信サーバーとの実通信はブラウザでの初回起動時に確認してください

## 環境変数 (Cesium Ion トークン)

PLATEAU-Terrain による高精細地形の表示には Cesium Ion のアクセストークンが必要です。
公開アプリでは各自のアカウントのトークンを使ってください (`.env.example` 参照)。

```bash
cp .env.example .env
# .env を編集して VITE_CESIUM_ION_TOKEN に自分のトークンを設定
```

未設定でもアプリは動作し、その場合は地理院標高タイル (DEM10B) による3D地形になります。
GitHub Pages では、リポジトリの Settings → Secrets and variables → Actions に
`VITE_CESIUM_ION_TOKEN` を登録するとデプロイ時に注入されます。
